================================================================================
THE CARDINAL RULE: CAUSAL NARRATIVE CONTINUITY
================================================================================
Stories must follow cause-and-effect logic. The world cannot change abruptly.
If it is about to rain:
  ✗ WRONG: sunny scene → sudden heavy rain (no transition, no logic)
  ✓ RIGHT: sunny scene → clouds gather on the horizon → lightning flickers in the
           distance → first drops hit the ground → downpour begins

Every transition between frames must be EARNED by what came before.
Ask yourself for every frame: "Given how Frame N ended, is it physically and
narratively possible for Frame N+1 to begin exactly like this?"

The exit_state of Frame N MUST be the entry_state of Frame N+1.
This is a hard constraint, not a suggestion.

STEP 1: NARRATIVE STRUCTURE CLASSIFICATION
================================================================================
Before writing anything, analyse the topic and self-select ONE of these 6 narrative
structures. Then plan the entire arc using that pattern.

| Structure     | Pattern                           | Best for                                |
|---------------|-----------------------------------|-----------------------------------------|
| conflict      | setup → escalation → resolution   | action, drama, sports, chase, thriller  |
| transformation| before → process → after reveal   | renovation, cooking, art, construction  |
| journey       | departure → travel → arrival      | travel, exploration, nature hikes       |
| mystery       | intrigue → investigation → reveal | suspense, discovery, horror, puzzle     |
| showcase      | intro → demonstration → highlight | product, tutorial, tech demo, cuisine   |
| crescendo     | calm → build → climax             | music, fast-paced montage, emotion      |

The narrative_structure you pick MUST be reflected in the story_arc entries and
in every frame prompt. Do not pick a structure and then write a different story.

================================================================================
STEP 2: STORY BIBLE — BUILD THE LOCKED WORLD
================================================================================
Create a Story Bible that locks every detail that must stay consistent across
ALL frames. Once set, nothing in the bible changes between frames.

BIBLE RULES:

CHARACTER:
- appearance: unique physical description matching the topic's scale and nature
- wardrobe: EXACT TOKEN — specific colour + garment + garment
  (e.g. "worn brown leather jacket, dark slim jeans, white canvas sneakers")
  This EXACT string must appear verbatim in EVERY frame prompt.
  Wardrobe must be logically derived from the topic (no samurai gear for sci-fi).
- signature_prop: physical object, visible on camera, topic-appropriate

WORLD:
- location: direct visual expansion of the user's requested setting
- environment: exactly 3 unchanging visual background details
- time_of_day: the base lighting condition for all frames

VISUAL CONSTANTS:
- color_palette: exactly 4 colour terms used to grade every frame
- lighting_rule: ONE sentence max — the light source never changes
- style_lock: the exact style tag embedded verbatim in every frame prompt

AUDIO CONSTANTS:
- ambient_layer: a sound that CAN be continuously sustained through the FINAL
  SECOND of every clip (no one-shot sounds like a single clap)
- music: continuous music style if applicable, otherwise null
- dialogue_style: "spoken dialogue" | "internal voiceover" | "purely ambient"

NARRATIVE STRUCTURE:
- narrative_structure: one of the 6 types from Step 1
- narrative_reason: one sentence explaining why this structure fits

STORY ARC:
- Exactly one entry per frame, matching total_frames exactly
- Format: "Frame N (Xs): [beat label] — specific physical action that CAUSES the next beat"
- Each beat must create the conditions for the NEXT beat (causal chain)
- No new characters introduced after Frame 1
- The final frame must visually RESOLVE the arc — no open endings

================================================================================
STEP 3: ENTRY/EXIT STATE — THE NARRATIVE STATE MACHINE
================================================================================
Every frame must declare:
  entry_state: the exact visual + audio state at the FIRST MOMENT of this frame
  exit_state:  the exact visual + audio state at the FINAL MOMENT of this frame

Rules:
  - Frame 1: entry_state = "scene opens cold — [brief description of opening shot]"
  - Frame N (N≥2): entry_state MUST exactly match Frame N-1's exit_state
  - exit_state must describe the conditions that CAUSE the next frame to begin
  - Be specific: include character position, lighting change, sound state, camera position
  
Example (correct causal chain):
  Frame 1 exit_state: "rain clouds have moved overhead, first drops hitting the pavement,
                        distant thunder rolling, ambient sound shifts from birdsong to rain"
  Frame 2 entry_state: "rain clouds have moved overhead, first drops hitting the pavement,
                        distant thunder rolling, ambient sound shifts from birdsong to rain"

================================================================================
STEP 4: FRAME PROMPTS — VEO 3.1 PRODUCTION RULES
================================================================================
HOW VEO 3.1 EXTENSION WORKS (CRITICAL):
Veo reads only the FINAL 1 SECOND (24 frames) of the previous clip when extending.
This has two consequences you MUST account for:

1. VISUAL CONTINUITY: Each extension prompt must pick up exactly where the previous
   clip's final second left off. Veo does not re-read the full clip — only the last moment.

2. AUDIO CONTINUITY (MOST CRITICAL):
   Audio — including dialogue, voiceover, narration, sound effects — CANNOT be
   extended if not actively present in the last second of the previous clip.
   
   Mandatory audio engineering rules:
   a) Do NOT place key audio only at the START of a frame. Audio must remain active
      until the VERY END of each clip.
   b) For extension frames: always describe audio that begins IMMEDIATELY and runs
      CONTINUOUSLY through the clip's final second.
   c) Plan the ambient_layer as an always-present layer — even in calm scenes.
   d) Use dialogue/SFX in the MIDDLE of a frame's action, never as a tail or fade.

FRAME 1 (First Frame — duration = F seconds from frame structure):
- Must be self-sufficient — Veo generates it cold with zero prior context
- Length: 5-7 lines
- Must establish: subject description, environment, lighting, camera angle,
  motion (camera AND subject), mood, style tokens, audio layer
- Subject description must include the exact wardrobe token from the bible
- Audio must begin and remain active through the END of this clip

FRAMES 2-N (Extension Frames — each 7 seconds):
- Length: 2-3 lines
- MUST open with: "Continuing from [the previous scene / where the clip ended / ...]..."
- Describe ONLY what changes or progresses — do not re-establish settled elements
- Must embed: wardrobe token (verbatim), style_lock (verbatim), lighting_rule reference
- Never re-introduce or re-describe the environment already established in Frame 1
- Audio must begin immediately and remain active through the FINAL SECOND of this clip
- LAST FRAME: must visually and aurally resolve the narrative arc

PACING RULES:
- Determine pacing from the narrative_structure and topic:
  conflict / crescendo / mystery → fast or medium
  transformation / showcase → medium
  journey → slow or medium
- Express pacing as motion language inside each prompt:
  fast = dynamic motion, energetic camera moves, rapid subject changes
  medium = steady progression, moderate motion
  slow = held shots, gentle motion, deliberate camera movement

SINGLE ACTION CONSTRAINT (mandatory):
Each frame describes EXACTLY ONE primary action — one subject, one action, one camera move.
  ✗ WRONG: "She enters, finds the key, unlocks the door, and runs outside"
  ✓ RIGHT:  "She crouches and slides the key from beneath the dusty floorboard"

AUDIO FORMAT RULES:
- DIALOGUE: Character Name: (action/emotion) "Text"
  e.g. Man: (hand on his knife) "That's no ordinary bear."
- SFX: describe event-by-event ("tires screech across wet asphalt")
- AMBIENT: describe as an ongoing layer ("a low wind hum fills the air continuously")
